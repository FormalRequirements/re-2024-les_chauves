Goals

G.1 Context and overall objectives

G.1.1 The Bal 3000 Turbo Max Pro is a completly new approach on mailing and mailing box. It aims to fix traditionnal issues with mailing boxes and improve the experience for the end user.

G.1.2 The Bal 3000 Turbo Max Pro will allow the end user to get the content of the received mails at any given point in time and at any given place.

G.2 Current situation

G.2.1 The Bal 3000 Turbo Max Pro is currently at its prototype stage, solutions to accomplish the goals are yet to be decided.

G.3 Expected benefits

G.3.1 The first benefit granted by the Bal 3000 Turbo Max Pro is to receive the content of any mail without having to interact with the physical system.

G.3.2 Since the system will allow the user to avoid having to get his mails redirected, one induced benefit from the first benefit is a decrease in usage of paper, cardboard and all sorts of adhesive.

G.4 Functionality overview

Note
 Overview of the functions (behavior) of the system. Principal properties only (details are in the System book) (see Handbook).

G.4.1 The system shall read any standard A4 letter within any standard envelope.

G.4.2 The system shall recognig4ze and distinguish well-known packages format (e.g. Amazon/UPS deliveries).

G.4.3 Convert the content of read letter and sent it by email to the user.

G.4.4 Allow user to register a set of 5 emails maximum.

G.4.5 The system should be able to send a notification when his capacity is full or near full (80%).

G.4.6 The system shall be open to be emptied by the user at any given time.

G.5 High-level usage scenarios

G.5.7 The system shall read any standard A4 letter within any standard envelope and send it by email to the user.

G.6 Limitations and exclusions

G.6.1 The system shall not be able to read any letter that is not a standard A4 letter.

G.6.2 The system shall send any email to any email address that is not registered by the user, including previous email adresses.

G.7 Stakeholders and requirements sources

G.7.1 Users The end user of the system, who will benefit from the system’s functionalities.

G.7.2 Letter carrier The person in charge of the delivery of letters.

G.7.3 Package carrier The person in charge of the delivery of packages.

E.2 Components

E.2.1 The system will need to be connected to the Internet to send emails.

  • E.2.2 The system will have a physical link to the Internet through an Ethernet cable.

  • E.2.3 The system will have to send requests to emailing servers/services (SMTP, POP, IMAP)

E.2.4 The system will need to be using a camera or scanner-like technology to read the content of the letters.

E.2.5 The system will need to have an interface (physical or software) to allow the user to register his email addresses

E.3 Constraints

E.3.1 The size of the Bal 3000 Turbo Max Pro is constraint by the law to of the following dimensions:

  • Dimensions minimales pour les habitations individuelles

    • Flap dimensions : 26 cm width * 3 cm height

    • Outer-box : 34 cm width * 26 cm height * 26 cm depth

  • Dimensions minimales pour les habitations collectives

    • Chaque compartiment doit pouvoir contenir un volume de 260 mm (largeur) x 260 mm (hauteur) x 340 mm (profondeur)
